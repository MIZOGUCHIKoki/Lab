    \subsection{コンピュータネットワークの発展}
    \begin{itemize}
        \item コンピュータの歴史
              \begin{itemize}
                  \item 1946年 ENIAC, 1949年 EDSAC
                  \item コンピュータは，当時はスタンドアローン\footnote{装置機器やソフトウェアを，他の機器やシステムには接続せずに単独で使用することで，孤立．}で利用されていた．
              \end{itemize}
        \item 通信の歴史
              \begin{itemize}
                  \item 1830年 モールス通信網
                  \item 1876年 電話の発明
              \end{itemize}
        \item コンピュータネットワーク（インターネット）の歴史
              \begin{itemize}
                  \item 1969年 米国で4大学のコンピュータを接続
                  \item 1983年 TCP/IP
                  \item 1984年 日本で東工大，慶應，東大のコンピュータを接続（300bps）．
                  \item 1990年 World Wide Web
              \end{itemize}
    \end{itemize}
    \subsection{コンピュータネットワークの分類}
    \begin{itemize}
        \item PAN (Personal Area Network)
        \item LAN (Local Area Network)
        \item MAN (Metropolitan Area Network)
        \item WAN (Wide Area Network)
    \end{itemize}
    \indent 他にもいくつかあるが，主に，LAN, WANが使われている．\par
    \textbf{規模・距離のみで分類されるわけではない．}
    \subsection{交換（Switchin）}
    \begin{itemize}
        \item 全ての通信相手と直接通信路を持つのは非現実的．一本の回線の通信を複数の通信で共有．
    \end{itemize}
    \dotfill\\
    \textbf{交換}\par
    少ない資源（回線）で任意の通信相手と接続することを可能にする仕組み．
    \begin{itemize}
        \item \textbf{回線交換方式}
              \begin{itemize}
                  \item 通信開始時に経路を設定，通信中は専有．
                  \item 実際に情報を送っていない時も専有．
              \end{itemize}
        \item \textbf{パケット交換}
              \begin{itemize}
                  \item 情報をパケットに分割して，宛先情報付きで送る．
                  \item 中堅地点では，その時の状況で適切に方へ送る．
                  \item 専有しないが遅延が発生，順序が保証されない．
              \end{itemize}
    \end{itemize}
    \subsection{コネクション（レス）型}
    \begin{itemize}
        \item 交換方式とはレイヤが異なる概念
              \begin{itemize}
                  \item \textbf{コネクション型}
                        \begin{itemize}
                            \item 実際のデータ送信前にコネクションを確率する方式．（TCP）確認応答の送受信がある．
                        \end{itemize}
                  \item \textbf{コネクションレス型}
                        \begin{itemize}
                            \item コネクションを確立せずに，各パケットに宛先を書く方式．（UDP, IP）
                        \end{itemize}
              \end{itemize}
    \end{itemize}
    \subsection{通信の形態}
    \begin{itemize}
        \item 通信の形態
              \begin{itemize}
                  \item ユニキャスト\\
                        1対1の通信（最近はこれ）
                  \item マルチキャスト\\
                        特定のグループ内のメンバーに送信（最近は流行らない）
                  \item ブロードキャスト\\
                        ある範囲（ネットワークセグメント）の全メンバーに送信
              \end{itemize}
        \item 通信の方向
              \begin{itemize}
                  \item 単方向（一方向のみデータを伝送）
                  \item 双方向\footnote{設定の自動化．周りの状況に合わせている．}
                        \begin{itemize}
                            \item \textbf{半二重} 双方向にデータ転送が可能だが，同時には一方向のみ．（トランシーバー）
                            \item \textbf{全二重} 同時に双方向データ伝送が可能．
                        \end{itemize}
              \end{itemize}
    \end{itemize}
    \subsection{階層プロトコル}
    \begin{itemize}
        \item プロトコル：通信を行うための約束事
              \begin{itemize}
                  \item 各社のプロトコルが乱立し，互換性がなくなると，他社コンピュータと通信するネットワークが構築できなくなる．
              \end{itemize}
        \item 国際標準化機構ISOが「OSI参照モデル」を提案\\
              OSI参照モデルは，プロトコルの整理の仕方．ネットワークは研究者が決めており，規格化されていない．
    \end{itemize}
    \subsection{OSI参照モデル}
    \noindent Open System Interconnection reference mode
    \begin{itemize}
        \item ISO が提唱これ自体はプロトコルでなく「参照モデル」
        \item 現在のインターネットに適切な分割ではない．
    \end{itemize}

    \begin{itemize}
        \item \textbf{7. アプリケーション層}\\
              アプリケーション毎のサービス提供機能
        \item \textbf{6. プレゼンテーション層}\\
              データの表現形式
        \item \textbf{5. セッション層}\\
              アプリケーション同士の会話制御
        \item \textbf{4. トランスポート層}\\
              両端コンピュータでの処理，服装抑制，信頼性提供
        \item \textbf{3. ネットワーク層}\\
              途中の中継処理，経路制御
        \item \textbf{2. データリンク層}\\

        \item \textbf{1. 物理層}\\
              データを物理的信号に変換，ケーブルやコネクタの形状．
    \end{itemize}

\section*{クイズ3}
\subsection*{問1}
IPアドレスに関する記述で正しのはどれか．
\begin{description}
    \item[\ci] IPv4のアドレスは32Bit．
    \item[\bt] IPv6のアドレスは64Bit．（正しくは128Bit）
    \item[\ci] IPv4ではネットワークアドレス部を任意のビット数に設定できる．
    \item[\ci] ブロードキャストアドレスは，IPアドレスのホストアドレス部を全て\verb|1|にしたアドレスである．
    \item[\ci] ネットワークアドレスは，IPアドレスのホストアドレス部を全て\verb|0|にしたアドレスである．
\end{description}
\subsection*{問2}
IPv4で，IPアドレスが\verb|222.229.64.8|のホストが接続されているネットワークのブロードキャストアドレスはいくつか？
ただし，サブネットマスクは20Bitとする．\\
\underline{解答}\par
サブネットマスクが20Bitなので，サブネットマスクの別表記は以下のようになる．
\begin{center}
    {\ttfamily 255.255.240.0}\\
    {\ttfamily 11111111 . 11111111 . 11110000 . 00000000}
\end{center}
\indent つまり，ネットワークアドレス部20Bit，ホストアドレス部12Bitから構成される．\par
この場合，ネットワークアドレス部は先頭から数えて17Bit - 24Bit目に着目して考える．その中で末尾4Bitの\(0,1\)の組み合わせは全部で\(2^4\)個あり，IPアドレス先頭から17Bit - 24Bit目の64Bitを1と数えて16個先のアドレスまでネットワークアドレスであることがわかる．\par
従って，このIPアドレスのネットワークアドレスは{\ttfamily 222.229.64.0 - 222.229.79.0}である．\par
その中でもブロードキャストは，ホストアドレス部が全て{\ttfamily 1}になる必要があるので，
\begin{center}
    {\ttfamily 222.229.79.255}
\end{center}
となる．
